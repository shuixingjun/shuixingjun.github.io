Call apply bind

  2015-12-07 • 分类: Font-End • 标签: javascript • 阅读量 次

apply、call两者区别

  1. 作用完全一样，但接受参数的方式不同。
  2. call需要把参数按顺序传递进去，而apply则把参数放到数组里。
  3. js中，某个函数的参数数量是不固定的，当参数明确知道数量时用call，而不确定时用apply，然后把参数push进数组传递过去。当参数数量不确定时，函数内部也可以通过arguments这个数组来遍历所有的参数。
  4. apply()第二个参数是一个数组或多个参数变量。如果第一个参数为null，那么this将指向全局对象，此时得到的结果就恰好如同当调用一个非执行对象时的方法。例如:
    var alien = {
sayHi: function (who) {
    Return "Hello" + (who ? ", " + who : "") + "!";
    }
};
alien.sayHi('world'); //输出"Hello, humans!"
sayHi.apply(alien,["humans"]);   //输出"Hello, world!"

常用方法：

1. 数组之间追加

var array1 = [12 , "foo" , {name "Joe"} , -2458]; 
var array2 = ["Doe" , 555 , 100]; 
Array.prototype.push.apply(array1, array2); 
/* array1 值为  [12 , "foo" , {name "Joe"} , -2458 , "Doe" , 555 , 100] */

2. 获取数组中的最大值和最小值

var  numbers = [5, 458 , 120 , -215 ]; 
var maxInNumbers = Math.max.apply(Math, numbers),   //458
    maxInNumbers = Math.max.call(Math,5, 458 , 120 , -215); //458

number 本身没有 max 方法，但是 Math 有，我们就可以借助 call 或者 apply 使用其方法。

3. 验证是否是数组（前提是toString()方法没有被重写过）

functionisArray(obj){ 
    returnObject.prototype.toString.call(obj) === '[object Array]' ;
}

4. 类（伪）数组使用数组方法

var domNodes = Array.prototype.slice.call(document.getElementsByTagName("*"));


Apply call bind 异同

var obj = {
    x: 81,
};

var foo = {
    getX: function() {
        return this.x;
    }
}
console.log(foo.getX.bind(obj)());  //81
console.log(foo.getX.call(obj));    //81
console.log(foo.getX.apply(obj));   //81

当希望改变上下文环境之后并非立即执行，而是回调执行的时候，使用bind（）方法，而apply/call则立即执行函数

总结：

  • apply 、 call 、bind 三者都是用来改变函数的this对象的指向的；
  • apply 、 call 、bind 三者第一个参数都是this要指向的对象，也就是想指定的上下文；
  • apply 、 call 、bind 三者都可以利用后续参数传参；
  • bind 是返回对应函数，便于稍后调用；apply 、call 则是立即调用 。
  • 当函数仅带有一个参数时，可以根据实际情况避免创建只有一个元素的数组工作，用call()比apply()更有效率。
