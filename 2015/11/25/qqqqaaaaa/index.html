基本值包装类型

javascript是一项面相对象语言，只有五种基本类型不是对象：数值类型，字符串类型，布尔类型，空类型和未定义得性，其中前三个类型有对应的以基本类型封装形式体现的对象表示。可以使用内置构造函数Number(),String(),Boolean()创建包装类型。
对象主要有两种类型：

  1. 是原生的native，包括内置对象（如数组和日期），以及用户自定义的对象。
  2. 是主机的host，包含windows对象和所有dom对象。

通常使用包装对象的原因之一，是有扩充值以及持久保存状态的需要。这是由于基本值类型并不是对象，他们不可能扩充属性。


尽量使用局部变量，而不是全局变量。

javascript有两个特性，1是可以使用变量不用声明，2是有暗示的全局变量。例如，在一个函数内有未经声明的变量，在调用该函数后，全局命名空间使用了另外的该变量，就会被覆盖住。另外一个原因是代码移植，如果代码运行在不同的环境，很可能导致不存在与原环境，但存在于其他环境的主机变量就被覆盖了。


变量释放的副作用。

通过var声明的全局变量，不可以用delete删除，而在函数内部的全局变量，则不能用delete删除。这就表明，隐含全局变量并不是真正的变量，而是全局对象的属性，属性可以用delete操作符删除，而变量不可以。


只使用1个var进行变量声明。


不增加内置构造函数（object()，array()，function()等）的原型，影响可维护性。

以下情形是例外：

  1. 未来可能将该功能作为一个统一的内置方法中。
  2. 自定义属性或方法未存在时。
  3. 进行文档记录。

给原型添加的属性在没有使用hasOwnProperty()时，可能会在循环中出现，导致混乱。

1. for-in循环
for (var i in man ) {
    if ( man.hasOwnProperty(i)) {
        console.log(i, ":", man[i]);
    }
}
2. 在Object.prototype中调用该函数
if (var i in man) {
    if (Object.prototype.hasOwnProperty.call(man,i)) {
        console.log(i, ":", man[i]);
    }
}


避免使用eval()

  1. eval()可以将任意字符串当做js代码执行。
  2. 包含一些安全隐患，有可能执行被篡改的代码。
  3. 影响到作用域链。
  • 使用parseInt(),可以从一个字符串中获取数值，最好不要忽略第二个参数，否则解析的字符串是0开始就会错误。
  • 使用JSON.parse()解析字符串，JSON.stringify()可以采用任意的对象或数组（或基本数据类型）,并且将其序列化为一个JSON字符串。


关于正则

两种方法创建正则表达式，优先使用字面量模式。
  1. 正则表达式的字面量
    var re = /\\/gm;
  2. 构造函数
    var re = new RegExp("\\\\", "gm");
字面量法第二个斜杠后，可将该模式改为不加引号的字母形式。
  • g—全局匹配
  • m—多行
  • i—大小写敏感的匹配

函数

js中函数有两个特征

  • 函数是第一类对象。
  • 他们提供局部作用域。

函数表达式

如果跳过函数表达式中的名称，将会得到一个未命名的函数表达式，即匿名函数。这并不影响函数的定义以及后续的调用，唯一的区别在于该函数对象的name属性将会成为一个空字符串。
就语法而言，函数表达式与函数声明很相似。
函数声明不需要用分号结尾，而函数表达式需要，且总是使用分号。

函数的模式

  1. API模式
    • 回调模式
      将函数作为参数传递
    • 配置对象
      有助于保持受到控制的函数的参数数量。
    • 返回函数
      当一个函数的返回值是另一个函数时。
    • curry化
      当新函数是基于现有函数，并加上部分参数列表创建时。
    何时使用curry化

    当发现正在调用同一个函数，并且传递的参数绝大多数都是相同的， 那么该函数可能是用于curry化的一个很好的候选参数。可以通过将一个函数集合部分应用到函数中，从而动态创建一个新函数。
    这个新函数将会保存重复的参数（因此，不必每次都传递这些参数），并且还会使用预填充原始函数所期望的完整参数列表。

    一些用法示例
     //普通函数
 function add(a, b, c, d, e) {
     return a + b + c + d + e;
 }

 //可运行于任意数量的参数
 schonfinkelize(add, 1, 2, 3)(5, 5); //16

 //两步curry化
 va addOne = schonfinkelize(add, 1);
 addOne(10, 10, 10, 10); //输出41
 var addSix = schonfinkelize(addOne, 2, 3);
 addSix(5, 5); //输出16
【转换函数schonfinkelize()并不局限于单个参数或者单步Curry化】

设计模式

1. 单体模式

保证一个特定类仅有一个实例。在js中，对象之间永远不会完全相等，除非它们是同一个对象，故即使创建一个具有完全相同成员的同类对象，它也不会与第一个对象完全相同。因此，可以认为每次在使用对象字面量创建对象的时候，实际上就正在穿件一个单体，并且并不涉及任何特殊语法。例如：

var obj = {
    myprop: 'my value'
};
var obj2 = {
    myprop: 'my value'
};
obj == obj2;  //false
obj === obj2; //false

使用new操作符

  var uni = new Universe();
  var uni2 = new Universe();
uni === uni2;  //true

在这个例子中，uni对象仅在第一次调用构造函数时被创建，之后创建时将会返回同一个uni对象。故uni与uni2本质上指向同一个对象的两个引用。实现这种模式有三种方法：

  • 全局变量存储该实例（不推荐）
  • 构造函数的静态属性中缓存该实例。
      function Universe() {
    //检查是否有现有实例
    if (type Universe.instance === 'object') {
        return Universe.instance;
    }
    //正常进行
    this.start_time = 0;
    this.bang = "Big";
    //缓存
    Universe.instance = this;
    //隐式返回
    return this;
  }
  //测试
  var uni = new Universe();
  var uni2 = new Universe();
uni === uni2; //true

    缺点：instance公开，故有可能在无意中修改instance属性，以至于实例丢失。

    • 将实例包装在闭包里。
      这种方法的秘诀在于重写构造函数。

    利用私有静态成员模式

    function Universe() {    
     //缓存实例
     var instance;
    //重写改构造函数
    Universe = function() {
        return instance;
    }    
    //保留原型属性。（这是由于自定义函数重写构造函数时会丢失所有在初始定义和重定义时刻之间添加到它里面的属性。）
     Universe.prototype = this;
    //实例
    instance = new Universe();
    //重置构造函数指针
    instance.constructor = Universe;
    //所有功能
     instance.start_time = 0;
    instance.bang = "Big";
    return instance;        
 }

    利用即时函数，第一次创建对象，使私有instance指向该对象，在第二次调用之后，构造函数仅返回该私有变量。 

    var Universe;
(function() {
    var instance;
    Universe = function Universe() {
        if(instance) {
            return instance;
        }
        instance = this;
        //所有功能
        this.start_time = 0;
        this.bang = "Big";
    }
})

2. 工厂模式

在函数内创建一个对象，给对象赋予属性及方法，再将对象返回，能够解决创建多个相似对象的问题。 

//父构造函数
function CarMaker() {}
//a method of the parent
CarMaker.prototype.drive = function() {
    return "Vroom,I have " + this.doors + " doors";
}
//静态工厂方法
CarMaker.factory = function(type) {
    var constr = type,
    newcar;
    //如果构造函数不存在 ，则发生错误
    if (typeof CarMaker[constr] !== 'function') {
        throw {
            name:"Error",
            message:constr + " doesn't exist"
        }
    };
    //在这里，构造函数是已知存在的，使原型继承父类，但只继承一次
    if (typeof CarMaker[constr].prototype.drive !== 'function') {
        CarMaker[constr].prototype = new CarMaker();
    };
    //创建一个新的实例
    newcar = new CarMaker[constr]();
    //可选择行的调用一些方法然后返回
    return newcar;
}
//定义特定的汽车制造商
CarMaker.Compact = function(){
    this.doors = 4;
};
CarMaker.Convertible = function(){
    this.doors = 2;
};
CarMaker.SUV = function(){
    this.doors = 24;
};
//实现
var corolla = CarMaker.factory('Compact');
var solstice = CarMaker.factory('Convertible');
var cherokee = CarMaker.factory('SUV');
corolla.drive(); // Vroom,I have 4 doors
solstice.drive(); // Vroom,I have 2 doors
cherokee.drive(); // Vroom,I have 24 doors
工厂模式使用场合
  1. 动态实现
    例如例子，创建一些用不同方式实现统一接口的对象，那么可以使用一个工厂方法或者简单工厂对象来简化实现过程。选择可以是明确进行的也可以是隐含的。
  2. 节省设置开销
    如果对象要进行复杂的并且彼此相关的设置的时候，那么工厂模式可以很显著的减少每种对象的代码量。将特定的设置代码提取出来会使得代码有极大地提升。并且能优化结构便于维护。
  3. 用于许多小型对象组成一个大对象。
  4. 工厂模式之利
    主要好处就是可以消除对象间的耦合，通过使用工程方法而不是new关键字。将所有实例化的代码集中在一个位子防止代码重复。
  5. 工厂模式之弊
    大多数类最好使用new关键字和构造函数，可以让代码更加简单易读。而不必去查看工厂方法来知道。

由于无从识别对象的类型，因为全部都是Object，不像Date，Array等，因此可以在内置的全局Object()构造函数。在这种内置对象工厂模式下，无论使用new操作符与否，都可以调用Object()。

var 0 = new Object(),
    n = new Object(1),
    s = Object('1'),
    b = Object(true);
//test
o.constructor === Object;    //true
n.constructor === Number;    //true
s.constructor === String;    //true
b.constructor === Boolean;    //true

3. 迭代器模式

对象需要提供一个next()方法。
hasNext()方法，该对象的用户可以使用该方法来确定是否已经到达了数据的末尾。
额外的便利方法：
rewind()，重置指针到初始位置
current()，返回当前元素，因为不可能在不前进指针的的情况下使用next()执行该操作。

4. 装饰者模式

动态添加福佳功能到对象中，一个比较方便的特征在于其预期行为的可定制和可配置。

  • 使每个装饰者成为一个对象，并且该对象包含应该被重载的方法。
  • 使用列表去实现。
    该方法不需要继承，并不是每个装饰方法调用链中前面的方法，可以简单讲前面的方法结果作为参数传递到下一个方法。

5. 策略模式

根据客户正在试图执行任务的上下文，从多个算法中选择用于处理特定任务的算法。可用于做表单验证。

6. 外观模式

为对象提供了一个可供选择的接口。

  • 两个或更多方法普遍的被一起调用时，创建一个方法以包装重复的方法调用。
  • 适用于浏览器的脚本处理，将浏览器之间的差异隐藏在外观之后。
    var myevent = {

    //IE 浏览器

  • 应用于重新设计和重构。例如：
    当需要替换一个具有不同实现的对象时，用外观模式，可首先考虑新对象api，然后继续在原有对象的前面创建一个外观。

7. 代理模式

一个对象充当另一个对象的接口。代理介于对象的客户端和对象本身之间，并且对该对象的访问进行保护。

  • 延迟初始化。通过替换本体对象的接口，解决应用程序实际并未使用客户端初始化对象的问题，只有在客户端明显需要本体对象完成一些工作时，代理才将两个消息一起传递。
  • 合并网络请求

8. 中介者模式

缓解紧耦合时改变单个对象的同时影响其他多个对象的问题，促进形成松耦合，且有助于提高可维护性。
在这种模式中，独立的对象之间并不直接通信，而是通过mediator对象，当其中的一个colleague对象改变状态后，它将会通知该mediator，而mediator将会把该变化传达到任意其他应该知道此变化的colleague对象。

9. 观察者模式

即自定义事件，订阅/发布模式。可促进形成松散耦合。


Dom和浏览器模式

1. 操作DOM

减少更新DOM，尽量分批处理。
当需要创建一个相对比较大的子树，应该在子树完全创建之后再将子树添加到DOM树中。可采用文档碎片技术来容纳所有节点。该技术在DOM树添加新结点时非常重要，在更新DOM现有部分时，仍然可以批处理提交修改：为需要修改的子树的根节点建立一个克隆镜像，然后对该克隆镜像做所有的修改操作，在完成修改操作后用克隆镜像替换原来的子树。

var oldnode = document.getElementById('result'),
    clone = oldnode.cloneNode(true);
//处理克隆对象...

//完成后
oldnode.parentNode.replaceChild(clone,oldnode);
